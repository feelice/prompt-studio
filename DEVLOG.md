# 开发日志 / Development Log — Prompt Studio

记录项目从想法到 v0.5 beta 的开发过程、关键决策、遇到的问题与解决办法。最新在最上方。

A narrative log of how the project evolved to v0.5 beta — decisions, problems, and fixes. Newest on top.

---

## 阶段六：提示词字典（→ v0.5 beta）

**目标**：把"整段固定模板"升级为"可自由拼装的积木"。

**设计决策**
- 数据结构 `DICT`：按字段（role/context/constraints/output）分组，每条片段带双语 `{zh,en}`，便于扩展。
- 接入形态：经与用户确认，选用「字段下方挂可点 chips」而非独立页面——与现有模板无缝结合，改动小。
- 拼接规则：图像类用 `, ` 连接，文本类按语言用 `；`/`; `；点选即追加、再点删除，并自动清理多余分隔符。
- 内容范围（首期全做）：通用（角色/上下文/约束/输出）、编程（开发·维护·测试规范）、图像（风格/画质/光线/镜头/构图/负向/MJ·SD 参数）。

**验证**：用 jsdom 真实 DOM 引擎自动化测试——确认点选写入字段、高亮状态、中英切换后 chips 仍在（绘图类 61 个 chip）。

## 阶段五：角色库（→ v0.4 beta）

**触发**：用户反馈"场景模板太少，很多主流角色没有"。

**动作**：基于 awesome-chatgpt-prompts（实采 209 个角色标题）精选，新增「🎭 角色库」类目，38 个主流角色（终端/解释器/各类老师/教练/编剧/各行业顾问等），场景模板总数升至 88。

**踩坑**：用 jsdom 测试时发现——导入角色类内容会因 `DEFROLE` 缺少 `roles` 键而报错（`Cannot read properties of undefined`）。已补键修复。健康/法律/财务/心理类角色统一加上"仅供一般参考，重要决策请咨询专业人士"的约束。

**经验**：每加一个新类目，要同步检查所有"按类目取值"的字典（DEFROLE、detectCat、importCat、CATKEYS、SPECS），否则会漏配导致运行时错误。

## 阶段四：版本号可视化 + 一次"乌龙"排查

**事件**：用户反映"场景只有 4 个"。排查发现两件事：
1. 早期 v0.1 的"目标类型"恰好是 4 个卡片，容易被误认为"只有 4 个选项"。
2. 用户截图里的 4 项（兄弟连、权力的游戏…）其实是**浏览器对搜索输入框的自动填充历史**，并非软件选项。

**解决**：
- 给搜索框加 `autocomplete="off"`，杜绝历史记录干扰。
- 顶部标题旁加**版本徽章**、浏览器标签也带版本号，方便用户一眼确认打开的是不是最新版。

**经验**：当用户描述的现象与代码逻辑矛盾时，先怀疑"打开了旧文件 / 浏览器行为"，并提供可自证的标识（版本号）。

## 阶段三：工程化功能（→ v0.3 beta）

新增本地保存/收藏（localStorage）、导出 promptfoo YAML（自动把 `{变量}` 转 `{{变量}}` 并生成断言骨架）、导出 DSPy 代码片段（Signature + ChainOfThought）。

**重大事件：工作区被重置**。某次会话间隙，挂载的项目目录与临时副本被清空。应对：用上下文中保存的完整源码**重建**全部文件，并写入可下载的输出目录。**经验**：及时把成果固化为可下载产物，并提醒用户保存到本机固定位置。

## 阶段二：双语化与大样本库（→ v0.2 beta）

- 引入 `I18N`，实现中英一键切换（界面、模板、输出、使用规范全双语）。
- 采集 GitHub 高星仓库，模板扩到 40+；新增搜索、下载 .md。

## 阶段一：最小可用版本（→ v0.1 beta）

- 单文件静态 HTML（HTML/CSS/JS 合一），纯前端、本地可用、数据不上传。
- 四大类目（通用大模型/编程/自动化/绘图）+ 三档输出（简版/标准/详尽）+ 各类目使用规范。
- 导入并定制：粘贴自然语言/粗糙 prompt → 启发式拆解为结构化模板；或生成"优化指令"交给 AI 精修。

---

## 技术约定 / Conventions

- **单文件**：所有 HTML/CSS/JS 内联在 `index.html`，零依赖、零构建、双击即用。
- **数据驱动**：内容集中在 `CATS`（类目）、`TPL`（模板）、`SPECS`（规范）、`DICT`（字典）、`I18N`（文案）几个对象，扩展只改数据。
- **测试**：交付前用 `node --check` 校验语法 + jsdom 跑真实 DOM 验证关键交互。
- **不联网**：所有处理在本地浏览器完成；"导入"用启发式拆解，复杂改写交由用户自己的 AI（meta-prompt）。
